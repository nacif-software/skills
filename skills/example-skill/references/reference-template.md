# Reference (template)

Put detailed material here — the stuff that would bloat `SKILL.md` if inlined.
The agent reads reference files **on demand**, so this is the right home for:

- Long checklists or rule sets (e.g. "40 things to verify before X").
- Lookup tables, naming conventions, or mappings.
- Worked examples and edge-case catalogs.
- Background that's useful occasionally but not on every invocation.

Keep `SKILL.md` pointing here ("see `references/reference-template.md`") instead of
copying the content into the skill body. That keeps the always-loaded part small
while still making the detail available when it's needed.

Delete this file (or replace it with your own references) when you build a real skill.
